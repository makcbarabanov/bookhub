## Фаза 2 — «Советы ИИ» (ближайший релиз) ✅ v0.4

**Цель:** кнопка 🤖 в панели → вкладка «Советы ИИ» → структурированный отчёт по книге.

### Продукт

- Анализ **только по кнопке** «🔄 Обновить анализ» (не на каждый клик)
- Кэш в БД — повторное открытие мгновенное
- Контекст: все главы + чек-лист + герои
- Категории (вкладки UI):
  1. **Неточности** — язык, логика, сюжет; кнопка «Применить» с diff (только безопасные правки)
  2. **Развитие сюжета** — идеи, куда вести линии
  3. **Кардиограмма сцены** — tension / pacing / atmosphere (радар по книге или топ-глав)

### Техника

- **OpenRouter** (Text), ключ в `.env` — `OPENROUTER_API_KEY`, модель настраиваемая
- Миграция `005_book_analyses.sql` — кэш JSON
- `GET` кэша + `POST` refresh + `POST` apply fix
- Лог токенов (`ai_usage_log`), лимит refresh/час
- Fallback: показать старый кэш при ошибке API

### Не в v1

- Авто-анализ при открытии вкладки
- Полная автозамена глав без подтверждения

## Фаза 3 — текстовый ассистент «Секретарь»

**Цель:** диалог в вебе (без голоса) с **действиями** в БД.

Примеры команд:

- «Добавь заметку в книгу 14:45: …»
- «Добавь идею в чек-лист»
- «Какая активная глава?»

### Техника

- LLM + function calling на бэкенде
- Tools: `list_books`, `add_checklist_note`, `add_inbox_note`, …
- Таблица `book_inbox_notes` — входящие заметки до переноса в чек-лист/главу

## Фаза 4 — голос в вебе (PWA)

**Цель:** кнопка 🎤 → говоришь → тот же API, что «Секретарь».

### OpenRouter

| Категория | Назначение |
|-----------|------------|
| **Transcription** | Голос → текст |
| **Text** | Понимание намерения + ответ |
| **Speech** | Ответ ассистента голосом (опционально) |

Альтернатива MVP: браузерный `SpeechRecognition`, OpenRouter только для текста.

## Фаза 5 — виджет на смартфоне (конечное видение)

**Сценарий Макса:**

1. Виджет на экране телефона
2. Кнопка «Голосовой набор»
3. Диалог с ИИ-помощником в реальном времени
4. Команда: *«Напиши заметку в мою книгу 14:45 …»*
5. Запись в PostgreSQL → сразу видно в `book.islanddream.ru`

### Варианты реализации

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **PWA** на том же домене | Один код с вебом | iOS ограничивает фон и микрофон |
| **Нативный мини-апп** | Надёжный виджет, push | Отдельная разработка |

### UX-принципы

- По умолчанию — **активная книга**; «в 14:45» — если название сказано явно
- Опасные действия (правка главы) — подтверждение голосом «да/нет»
- Заметки сначала в **inbox**, не сразу в тело романа
- Офлайн: запись аудио → транскрипция при появлении сети (позже)

## Фаза 6 — масштабирование ИИ

- **Embeddings** + **Rerank** — когда книга не влезает в один запрос (поиск по фрагментам)
- Локальная модель (**Ollama**) как fallback при стоимости OpenRouter
- Не путать: **Whisper** = речь→текст; для текста романа — **Text**-модели
